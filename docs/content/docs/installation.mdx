This guide covers installing the ZAP runtime, code generator, and compiler.

## Prerequisites

- .NET 6.0 or later (recommended) or .NET Standard 2.0 compatible runtime
- ZAP compiler (`zap`) for schema compilation

### Installing the ZAP Compiler

The ZAP compiler is required at build time to compile `.zap` schema files.

**Windows (Chocolatey):**

```bash
choco install zap
```

**macOS (Homebrew):**

```bash
brew install zap
```

**Linux (apt):**

```bash
sudo apt-get install zap
```

**Linux (from source):**

```bash
curl -O https://zap.dev/zap-cpp-1.0.2.tar.gz
tar zxf zap-cpp-1.0.2.tar.gz
cd zap-cpp-1.0.2
./configure
make -j6 check
sudo make install
```

Verify installation:

```bash
zap --version
```

## NuGet Packages

### Runtime Package

The runtime package is required for all ZAP operations:

```bash
dotnet add package Zap.Net.Runtime
```

Or via Package Manager Console:

```powershell
Install-Package Zap.Net.Runtime
```

### MSBuild Integration (Recommended)

For automatic code generation during build:

```bash
dotnet add package ZapC.CSharp.MsBuild.Generation
```

This package:
- Automatically detects `.zap` files in your project
- Generates C# code during build
- Supports incremental builds

Add schema files to your `.csproj`:

```xml
<ItemGroup>
  <Zap Include="schemas/*.zap" />
</ItemGroup>
```

### Command-Line Generator

Install the code generator as a global .NET tool:

```bash
dotnet tool install zapc-csharp --global
```

Usage:

```bash
zap compile -o csharp schema.zap
```

## Project Setup

### Minimal .csproj

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Zap.Net.Runtime" Version="1.*" />
    <PackageReference Include="ZapC.CSharp.MsBuild.Generation" Version="1.*" />
  </ItemGroup>

  <ItemGroup>
    <Zap Include="**/*.zap" />
  </ItemGroup>
</Project>
```

### Directory Structure

Recommended project structure:

```
MyProject/
  MyProject.csproj
  Program.cs
  Schemas/
    addressbook.zap
    calculator.zap
```

## Verifying Installation

Create a test schema `test.zap`:

```protobuf
@0xabcdef1234567890;

struct TestMessage {
  text @0 :Text;
  number @1 :Int32;
}
```

Build your project:

```bash
dotnet build
```

If successful, `test.zap.cs` will be generated in your `obj` directory and compiled into your assembly.

## Troubleshooting

### zap not found

Ensure the ZAP compiler is installed and on your PATH:

```bash
which zap  # Unix
where zap  # Windows
```

### Schema compilation errors

Check that your schema file has a valid unique ID:

```protobuf
@0x1234567890abcdef;  # Generate with: zap id
```

### MSBuild not generating code

Verify the package is installed and schema files are included:

```bash
dotnet list package
```

Check that `<Zap Include="..." />` items exist in your project file.
